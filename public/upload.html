접수일 기준 클레임 데이터

ERP에서 접수일 기준으로 다운받은 엑셀 파일을 업로드합니다.
claims_receipt 테이블에 저장되며, 요구내역 컬럼은 자동으로 증상 파싱 파이프라인을 거칩니다.
미분류(미매칭) 항목은 symptom_pending 테이블에 별도 저장되어 관리자 검토를 기다립니다.

필수 컬럼: 접수번호 · 접수일(YYYY-MM-DD) · 제품군 · 유형
선택 컬럼: 판정유형 · 하자유형 · 모델 · 국가 · 비용 · 요구내역 · 판정결과 · 대분류 · 소분류 · 불량부위 · LOT · 코드 · 브랜드
※ 요구내역(claim_detail)은 증상 자동 파싱에 사용됩니다. 접수번호 기준 중복은 첫 번째 행이 유지됩니다.

월별 매출량 데이터 (KPI 불량지수용)

KPI 현황의 불량지수(%) = 클레임건수 / 매출건수 × 100 계산에 필요한 매출량 데이터입니다.
sales_monthly 테이블에 저장되며, 같은 월+국가 데이터가 있으면 자동으로 업데이트됩니다.

필수 컬럼: year_month(연월, YYYY-MM) · country(국내/베트남) · sales_count(매출건수)
선택 컬럼: sales_amount(매출액, 원 단위)
※ 엑셀 예시: 2025-01 | 국내 | 19188 | 842000000

월별 실패비용 데이터

KPI 현황의 실패비용 현황에 사용됩니다.
failure_costs 테이블에 저장되며, 같은 연월 데이터가 있으면 자동으로 업데이트됩니다.

필수 컬럼: year_month(연월, YYYY-MM) · revenue(매출액, 원) · haza_product(제품) · haza_material(자재) · haza_t50re(T50RE 헤드자재) · haza_t50re_labor(T50RE 헤드인건비) · haza_claim(클레임보상) · baros_contact(AS컨택센터) · baros_as(AS조치비) · baros_logistics(AS물류비)
컬럼명 자동 매핑: 한글 헤더도 인식합니다 (예: "매출액" → revenue, "제품" → haza_product)

회수일 기준 클레임 데이터 claims 테이블

ERP에서 회수일 기준으로 다운받은 엑셀 파일을 업로드합니다.
claims 테이블에 저장되며, KPI 자동 분석 및 제품별 클레임 현황에 사용됩니다.
필수 컬럼: 접수번호 · 순번 · 회수일자 · 판정형태 · 제품구분 · 브랜드 · 회사

컬럼 매핑: 접수번호+순번 → claim_id  ·  회수일자 → claim_date  ·  판정형태 → category  ·  판정구분 → judgement_type  ·  하자유형 → defect_type  ·  제품구분 → item  ·  제품코드 → model  ·  금액 → cost  ·  요구내역 → claim_detail  ·  판정결과 → inspection_result  ·  부품 → defective_part  ·  LOT → lot  ·  브랜드 → brand  ·  회사 → country (베트남 포함 시 → '베트남', 그 외 → '국내')
